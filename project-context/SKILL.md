# Project Context (项目上下文常驻管理器)

## Core Purpose (核心目标)
此技能的核心思想是：**用强校验的文件替代 AI 记忆**。它维护一份始终最新、且经过防伪验证的 `.ai/project.md`，作为整个项目的持久化"大脑"。任何技能开始前，都必须先读取该文件以建立世界观。

**为了解决 AI 产生幻觉的问题，此文档引入了「严格实体校验」和「人工审查红绿灯」机制。**

---

## 核心防伪与门控机制

所有操作该文件的 AI 代理必须遵守以下刚性法则：
1. **实体存在法则 (Single Source of Truth)**：任何写入 `project.md` 的代码文件路径，写入前必须通过文件系统验证其真实存在。严禁捏造路径。
2. **状态机门控法则**：文件的 frontmatter 中包含 `status`。
   - `status: draft`：当前内容尚未经过人类验证。AI 可以生成并设置为此状态。
   - `status: approved`：**人类专属特权**。只有人类可以修改为 approved。AI **严禁**将状态修改为 approved 或移除 draft。
3. **防篡改法则**：如果文件已经是 `approved`，AI 只能做**局部增量追加**（如新增 ADR 或功能状态），严禁大面积重写覆盖已通过审核的基础架构描述。

## 工作模式

### Mode 1 — INIT（项目初始化）

**触发条件**：用户要求"初始化项目上下文"，或检测到 `.ai/project.md` 不存在。

**执行步骤**：
1. **探测基本信息**：扫描 `package.json`/`go.mod` 等识别技术栈。只扫描根目录和一层子目录，避免 token 爆炸。
2. **实体路径校验 (Critical)**：列出核心入口点或配置文件，**必须使用命令行或内置工具检查路径是否存在**。
3. **生成初版文档**：严格按照下方《标准模板》生成，并强制加上 `status: draft` 的 frontmatter。
4. **交出控制权 (Handoff)**：明确告知用户：“项目初始上下文已生成（草稿状态）。请人工核验关键模块路径，确认无误后，请手动将其改为 `status: approved`”。

### Mode 2 — READ (任务前加载)

**触发条件**：被任何下游 Agent (如 system-architect / task-coder) 调用，或流水线刚开始。

**执行步骤**：
1. 使用 `view_file` 读取 `.ai/project.md`。
2. **状态预警**：如果文件不存在，触发 INIT。如果存在但状态是 `draft`，你**必须**输出一条醒目的警告：“⚠️ 当前项目上下文仍为 Draft 草稿状态，可能存在未验证的幻觉描述，请使用者注意风险！”并继续执行。
3. 简单总结摘要（≤5句话），并通告后续节点（如 task-coder / system-architect）可以基于此展开工作。

### Mode 3 — UPDATE (任务后更新与漂移检测)

**触发条件**：完成一个业务功能开发后，或发生重大架构变动后。

**执行步骤**：
1. 第 0 步：**漂移检测 (Drift Check)**。读取文件中记录的关键路径，抽样检查是否在本地依然存活。如果发现大面积 404，输出 `[DRIFT_DETECTED]`，要求人类重新审计。
2. **精准局部追加**：不允许全量覆写。使用精准内容替换：
   - 勾选进度：将「进行中」移至「已完成」。
   - 新增模块：追加一行表格数据（必须先验证路径）。
   - 架构调整：追加一行到 ADR（架构决策记录）。
3. 追加变更日志（Changelog）。

## Standard Template (`.ai/project.md` 模板)

```markdown
---
status: draft                # 必须是 draft，留给人类改成 approved
last-verified: YYYY-MM-DD
---
# [项目名称]

> **一句话定位**：[项目是用什么语言写的什么系统，解决什么问题]

## 技术底座 & 全局规约 (Conventions)
- **语言/框架**：[如 React + TypeScript + Go]
- **架构模式**：[如 Monorepo 划分，目录规范]
- **编码铁律**：[此处提取原 conventions.md 的浓缩版，如国际化要求、状态管理工具]

## 关键模块实体镜像 (已校验路径)
| 模块代号 | 实体物理路径 (必须存在) | 核心职责 |
|---------|----------------------|---------|
| [名称] | [相对路径] | [简述] |

## 功能演进状态
### ✅ 交付完成
- [模块名]: [简要说明]
### 🚧 当前聚焦
- [模块名]: [简要说明]

## 架构决策库 (ADR)
| 日期 | 决策事件 | 核心动因 (Why) |
|------|---------|---------------|
| YYYY-MM | [采用了XX方案] | [解决了什么难题] |

## 外部契约与红线
- ⚠️ 禁区：[绝对不允许破坏的接口或调用关系]
```
