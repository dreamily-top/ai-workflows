# Test Runner (自动化测试探测与执行器)

## Core Purpose
此技能负责在 task-coder 编码完成后，自动运行项目测试并验证代码逻辑，确保每一次功能增量都能通过持续集成的标准检测。

## Context & Components
本技能依赖于同级目录下的探测脚本 `scripts/run_tests.js`。你不需要猜测项目环境，脚本会自动侦测 Node.js, Go, Python 或 Java 测试配置并执行下发。

## Instructions
当你被要求验证、测试或确认刚才的代码没有改坏时，请严格遵守以下步骤：

1. **调用脚本探测与执行**：运用命令行工具运行本技能所在目录下的 `scripts/run_tests.js` 来获取底层真实输出（绝不可自己捏造测试结果）。
2. **解读分析结果**：
   - 如果测试套件返回绿色 Pass 状态：确认改动安全，可推进给下一个审核环节。
   - 如果发生编译 / 测试 / 类型失败（Fail）：提取报错日志的堆栈（Stack Trace）。
3. **回旋修正 (Self-Healing)**：将抓取到的核心错误日志与失败文件，直接派发给 `task-coder`，强制指令它进行定位和代码修复。
4. **触发熔断防护**：如果同一次修复循环失败超过 3 次，立即停止该循环并对外输出 `[BLOCKED]` 原文，等待人类开发者介入。

## Standard Output Format
分析结束后，向用户或协作 Agent 进行如下总结汇报格式：
```
✓ 自动化测试探测: [通过 / 未通过]
- 耗时与状态: [简单概括]
- 待执行动作: [走向 Reviewer / 发回重写]
```
